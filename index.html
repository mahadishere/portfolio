SolidVR

Official Release

         

Project goal

    
Optimization of the Data Transfer Process from CAD System to VR-Application in Real-Time Via Direct Interface
    

Product Output

    
CAD data is often imported in most VR visualization solutions using an exchange format (e.g. STP or FBX). It takes a lot of time to export from the CAD system and import it into the visualization solution. Thus, created an interface between two systems and investigated different optimization methods.

Video Demo on Youtube

         
This product optimizes the pre-surgical planning through the power of Hololens 2 for hip replacement surgeries. Dicom Measurement Tools

Video Demo on Youtube

Note: This video demo was created with an old version of this application. In the newer version the rendering is more detailed, fast and less pixelated. See the images below for latest version

     
This app facilitates multiple measurement tools for Dicom images. Such as:
  • Angle, scale, cobbs angle, point, orientation, elipse, circle, etc.
  • Calculations are done with respect to the original resolution of the image.
  • Dicom Images are adjusted with respect to the screen resolution but their own resolution is maintained.
  • Lines and points can be deleted, moved, adjusted, etc.
Specially designed for carrying out pre-surgical plans.
         

AprilTag 3 for Unity

Video Demo on Youtube

Note: Special thanks to Keijiro for creating the AprilTag support for unity.

This project is an upgrade of the already available repo of AprilTag 2 for unity by Keijiro. Without the help from this repo it would be really hard to make this project work. In the video demo, the right side presents the Keijiro's version and the left side presents the version with the latest AprilTag 3 DLL. This application was created to test differences between Keijiro's DLL of AprilTag for unity and the one created with latest version of AprilTag 3 from the official repository.

Following are some of the differences that were found during the testing:

  • The version with the latest AprilTag 3 DLL works faster and more robust given any exposure level of the light
  • Keijiro's version doesn't support pthread, on the contrary AprilTag 3 DLL supports multithreading using pthread of c++
  • Keijiro's version doesn't support arm64 universal windows platoform support
   This is a project in prototyping phase with the goal to make it work for Hololens 2.
